# Handoff: 書架＋年表 — 京都古典籍 全文検索（Pattern B / 藍鉄ナイト）

## Overview（概要）
歴史系の若手研究者を主対象とした、京都古典籍カタログ（全2,716件・約4万画像）の探索 UI。
SNS 感覚で「書庫を雑見」する俯瞰体験と、目的の一件への最短到達を両立させる。
中核は2つの空間メタファ:

1. **年表リボン** — 12〜19世紀の時間軸。資料を点でプロットし、世紀クリック／帯ドラッグで年代を絞り込む。
2. **テーマ別の書架** — 「境内をめぐる」「洛中洛外を俯瞰する」等、主題ごとの横スクロール棚（Netflix/Spotify 的）。

ここから資料（本）へドリルダウンし、見開き単位で AI 構造化された翻刻・固有表現・原画像を閲覧する。

採用配色は **藍鉄ナイト（INDIGO NIGHT）** に確定。古典籍の図版を暗い面で際立たせる近未来トーン。

---

## About the Design Files（このバンドルの位置づけ）
このフォルダの HTML/CSS/JS は **デザインのリファレンス（意図した見た目と挙動を示すプロトタイプ）** であり、
そのまま本番に流用するコードではない。タスクは、これらのデザインを **対象コードベースの既存環境（React / Vue / Svelte 等）と既存の実装パターン・ライブラリで再現すること**。
環境がまだ無い場合は、プロジェクトに最適なフレームワークを選定して実装する。

プロトタイプは依存なしのバニラ JS で書かれている（フレームワーク選択を縛らないため）。
実装時はコンポーネント分割・状態管理・データ取得を対象環境の流儀に合わせること。

## Screens / Views（画面構成）

### 1. ヘッダー（sticky, 全幅）
- 高さ ≈ 57px。`backdrop-filter: blur(14px)`、半透明地 `--topbg`、下罫 1px `--line`。`z-index:40`。内側 `max-width:1480px` 中央寄せ、padding `11px 22px`。
- **ブランド**: グロー付きドット（8px 円, `--diagram`, `box-shadow:0 0 10px`）＋「京都古典籍」（明朝 1.05rem）＋補足ラベル「書架＋年表で巡る」（0.72rem, `--muted`, モバイル非表示）。
- **検索バー（コマンドバー）**: 角丸 pill（999px）, 幅 max 520px, 左に ⌕ アイコン。フォーカスで枠 `--diagram`＋3px グローリング。
  - フォーカス／入力で下に **サジェストドロップダウン**（角丸14px, `--sh-lg`）。グループ: 「全文・翻刻を検索」「建造物」「資料」「年代」。各項目は左にカラータグ（種別色）、資料は右に「N画像」。
  - 選択時の遷移: 全文→`q` 設定 / 建造物→建造物フィルタ / 資料→**資料ビュー**へ / 年代→その世紀の era を設定。

### 2. 年表リボン
- `max-width:1480px`、見出し「年表で巡る」（明朝）＋説明文＋右端「年代の絞り込みを解除 ✕」（絞り込み時のみ表示）。
- **トラック**: 高さ 96px, 角丸14px, 縦グラデ `--card→--bg2`, 1px枠。`cursor:crosshair`。
- **世紀目盛**: 12〜19世紀。各世紀の中央（=(c-1)*100+50 年）に tick＋ラベル。位置 = `(year - 1100) / (1900-1100) * 100%`。クリックでその世紀 `[(c-1)*100, c*100]` を era にトグル。
- **資料ドット**: 各資料を刊年位置にプロット。13px 円, 枠2px `--card`。区分「絵図・地図」は `--diagram`、他は `--accent`。hover で 1.5倍＋グロー、上に明朝フラグ（資料名・西暦）。クリックで**資料ビュー**へ。era 範囲内のドットは `--ink` で塗り＋1.6倍。
- **ドラッグ範囲選択**: トラックの空白を pointer ドラッグ → 10年スナップで `era=[min,max]`（幅≥10年で確定）。ドラッグせずクリック（=空白タップ）で era 解除。選択帯 `.tl-band` は `--diagram` 14%地＋左右1.5px境界。
- **モバイル(≤820px)**: トラック／ドット／帯は非表示にし、横スクロールの **世紀チップ**（「すべての年代」＋12〜19世紀）に置換。選択チップは `--diagram` 地。

### 3. 適用フィルタ・バー
- era / 建造物 / 全文クエリ が有効なとき、「絞り込み中:」＋チップ群を表示。各チップ（pill, `--diagram` 14%地）クリックでその条件を解除。

### 4. 書架（shelves）
- テーマ別セクションを縦に列挙（データの `shelves`）。各棚: 見出し（明朝 1.05rem）＋「N 点」＋3点超で「すべて見る ›」ボタン。
- **rail**: 横スクロール flex, gap 12px, `scroll-snap-type:x mandatory`。末尾に破線 endcap「この棚をすべて見る ›」。
- **「すべて見る」展開**: 棚に `.open` を付与 → rail が `display:grid`（`repeat(auto-fill,minmax(180px,1fr))`）に変化し全件をラップ表示、ボタンは「折りたたむ ▲」、endcap 非表示。
- ヒット0件の棚は `opacity:.3` に減光（消さない＝全体の地図感を保つ）。全棚0件なら空メッセージ。
- **資料カード（bcard）**: 幅188px（展開時 auto）, 角丸12px, `--card`地, 1px枠。
  - 画像: アスペクト 3/4。種別バッジ（種別色地, `--badge-ink`文字, 0.6rem, 600）。要約（明朝0.8rem, 2行クランプ）。メタ（資料名＋刊年西暦, 0.66rem）。
  - hover: `translateY(-3px)`＋`--sh-lg`＋枠が `--diagram` 寄り。クリックで詳細モーダル。

### 5. 資料ビュー（本へのドリルダウン）
- 書架を隠して表示。「‹ 書架へ戻る」ボタン。
- **ヘッダーカード**: 表紙画像（96px, 3/4）＋区分バッジ＋資料名（明朝1.35rem）＋読み（0.78rem）＋書誌メタ（著者/NDC/請求記号/刊年/全画像数）＋解説（明朝, `--ink2`）。
- **見開きグリッド**: その資料の構造化済み見開きを `repeat(auto-fill,minmax(180px,1fr))` で全件表示（bcard と同形）。クリックで詳細モーダル。

### 6. 詳細モーダル（見開き）
- 中央ダイアログ（max 900px, 角丸18px, `--sh-lg`）。地は `--scrim`＋blur。グリッド `340px 1fr`（モバイルは1列・全画面）。
- **左=画像**: 地 `--imgbg`, `object-fit:contain`。左下に前後送りボタン ‹ ›、右下に「現在位置 / 総数」（例 1/28）。
- **右=本文**: 閉じる×（右上円ボタン）。メタ行（種別バッジ／出典バッジ「AI構造化」or「NDL翻刻」／年号／確信度%）。見出し（要約の先頭文, 明朝1.3rem）。要約（明朝, `--ink2`）。
  - **注記・固有表現**: pill 群。種別で色分け（建造物=`--diagram` / 地名=`--accent` / 人名=`--ent` / 年号=`--accent2` / 注記=`--muted`）。背景はその色を `color-mix` で 14%, 枠 40%。クリックで建造物/地名/注記はフィルタ化しモーダルを閉じる。
  - **翻刻テキスト**: 明朝, line-height1.9, `--bg`地の枠付きブロック, `white-space:pre-wrap`。検索語は `<mark>`（地 `--mark`）でハイライト。
  - **資料行（recline）**: 資料名＋区分/NDC/画像数＋「この資料を見る ›」。クリックで資料ビューへ。
  - 「原資料を高精細で開く ↗」リンク（large 画像 URL を別タブ）。
- **キーボード**: `←/→`=前後送り, `Esc`=閉じる, `/`=検索フォーカス。

### 7. 配色スイッチャー（プロト専用 / 本番任意）
- 右下固定パネル。5案を切替（藍鉄ナイト＝確定。群青ネオン/墨翠/白磁クローム/生成りセピアは探索の記録）。
- **本番では藍鉄ナイトのみ実装すれば足りる。** ダーク/ライト切替を提供したい場合のみ多テーマ機構（`html[data-theme]` で CSS 変数を差し替え, 選択を localStorage 保存）を採用する。

## Interactions & Behavior（挙動）
- **検索**: 入力で即フィルタ（全文・要約・固有表現・資料名を部分一致）。サジェストは focus/input で表示、外側クリックで閉じる。Enter で全文確定。
- **年表**: 世紀クリック＝トグル絞り込み、ドラッグ＝範囲選択（10年スナップ）、ドット＝資料ビュー。
- **棚**: カード=詳細モーダル、「すべて見る」=その棚をグリッド展開。
- **モーダル前後送り**: 開いた文脈の見開き集合（棚一覧 or 資料内）を順送り。位置表示あり。
- **遷移**: テーマ切替は `background-color .35s`。カード hover `.15s`。`prefers-reduced-motion:reduce` で全アニメ無効。
- **フォーカス可視**: `:focus-visible` で `--accent` 2px アウトライン。カードは `tabindex=0`＋Enter 起動。

### Responsive（レスポンシブ／デバイス別操作感）
- ブレークポイント **820px**。
- **PC（>820px）**: 年表トラック＋ドット＋ドラッグ選択、棚は横スクロール、詳細は中央モーダル、キーボード操作。
- **スマホ（≤820px）**: 年表は世紀チップに置換、棚は横スワイプ、詳細モーダルは全画面（画像上 42vh）、配色パネルは下部横並び。
- 実装方針: 単なるリフローではなく、**入力系（hover/keyboard ↔ tap/swipe）と年表 UI（ドラッグ ↔ チップ）をデバイスで出し分ける**こと。

## State Management（状態）
```
state = {
  q:        string,          // 全文クエリ
  era:      [start,end]|null, // 選択年代（西暦）
  building: string|null,     // 固有表現フィルタ（建造物/地名/注記）
  record:   recordId|null,   // 資料ビュー表示中の資料
  open:     Set<shelfId>,    // 「すべて見る」展開中の棚
}
// モーダル用: navList(現在文脈の見開き配列), navIdx(現在位置)
```
- フィルタ合成: `pageMatch(page)` = q AND building AND era をすべて満たす見開き。
- `record` が非 null の間は書架を隠し資料ビューを描画。
- 年表・検索・固有表現クリックは互いに連動（例: 固有表現クリックで building 設定→書架に戻って絞り込み）。

### Data fetching（データ取得）
- プロトは `data.js` の静的データ。本番は検索 API（全文＋ファセット）に置換する想定。
- 推奨エンドポイント像: `GET /search?q=&era=&building=&kubun=&ptype=&ndc=` → 見開き配列＋ファセット集計。資料単位 `GET /records/:id` → 書誌＋見開き一覧。
- 画像は提供元サーバを直接参照（下記 Assets）。サムネ(S)と高精細(M)の2サイズ。

### Typography
- 明朝（見出し・要約・翻刻）: `--serif = "Hiragino Mincho ProN","Yu Mincho","Noto Serif JP",serif`

- スケール: 資料名/見出し 1.05–1.35rem、要約 0.8–0.95rem、本文翻刻 0.95–1rem(lh1.9)、UIラベル 0.66–0.82rem、タグ 0.6–0.66rem。見出しは `letter-spacing:.04em`。
- ※ Hiragino は macOS 標準。Web 配布時は Noto Serif/Sans JP で代替。

### Radius / Shadow / Spacing
- 角丸: pill `999px` / カード `12px` / パネル・モーダル `14–18px` / バッジ `5px`。
- 影: `--sh = 0 1px 2px rgba(0,0,0,.4),0 8px 22px rgba(0,0,0,.45)` / `--sh-lg = 0 2px 8px rgba(0,0,0,.5),0 24px 60px rgba(0,0,0,.62)`。
- 余白基準: コンテナ `max-width:1480px`、左右 padding PC22px/SP13px、グリッド/列 gap 12px。
- ブレークポイント: `820px`。

## Data Model（`data.js` 参照）
- `records[]`: `{ id, title, yomi, kubun, author, ndc, call, images, year:{w(和暦),s(西暦)}, blurb }`
- `pages[]`（見開き）: `{ id, pkey, thumb, large, glyph, record, ptype, conf, src, sum, text, ents:[{t,kind}], year? }`
  - `ptype`: 平面図_指図 / 絵図 / 本文 / 目次 / 口絵 / 見返
  - `src`: 「AI構造化」(Claude vision) or「NDL翻刻」(NDL古典籍OCR-Lite, CC BY 4.0)
  - `ents.kind`: 建造物 / 地名 / 人名 / 年号 / 注記
- `shelves[]`: `{ id, title, kind, pageIds[] }`（テーマ別キュレーション）
- `facets`: kubun / ptype / ndc の集計。`buildings[]`: よく使う建造物タグ。`years[]`: 年表プロット用。
- 種別→色のマップ `ptColor(ptype)`: 図面/絵図→`--diagram`、口絵→`--accent2`、本文→`--accent`、その他→`--ent`。
